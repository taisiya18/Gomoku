# Gomoku (Пять в ряд) — Консольная реализация на .NET 9

## 1. Обзор проекта
Данный проект представляет собой полноценную реализацию классической стратегической игры **Гомоку** (Gomoku). Программа разработана как консольное приложение на языке C# и ориентирована на демонстрацию навыков объектно-ориентированного проектирования, работы с файловыми системами и использования современных возможностей платформы .NET.

Проект включает в себя не только игровую механику, но и развитую инфраструктуру: систему сохранений, таблицу рекордов и интерактивный пользовательский интерфейс.

## 2. Игровая механика и правила
Игра строго следует классическим правилам Гомоку:
* **Игровое поле:** Квадратная сетка размером **15x15**.
* **Процесс:** Два игрока поочередно выставляют свои камни (символы) на свободные клетки поля.
* **Цель:** Первым составить непрерывный ряд из **пяти** символов своего цвета по горизонтали, вертикали или диагонали.
* **Контроль ходов:** Система автоматически проверяет корректность каждого хода, запрещая попытки поставить символ в уже занятую клетку или за пределы игрового поля.

## 3. Технические особенности реализации

### Работа с данными и JSON
Одной из ключевых технических сложностей проекта стала реализация сохранения состояния игрового поля.
* **Кастомная сериализация:** Для корректной работы с двумерными массивами (`int[,]`), которые стандартные средства `System.Text.Json` не всегда обрабатывают оптимально, был разработан специализированный конвертер — **`TwoDimensionalIntArrayConverter.cs`**. Этот компонент обеспечивает бесшовное преобразование игрового поля в JSON-структуру и обратно без потери данных.
* **Долгосрочное хранение:**
    * **Состояние игры:** Текущая расстановка сил и параметры игроков записываются в JSON-файл, что позволяет прервать и продолжить партию в любое время.
    * **Таблица рекордов (Top-Scores):** Результаты лучших матчей сохраняются в отдельном файле, обеспечивая сохранность данных между сеансами работы программы.

### Пользовательский интерфейс (TUI)
* **Интерактивное меню:** Реализована удобная навигация, позволяющая вводить имена игроков, начинать новые партии, загружать сохранения и просматривать таблицу лидеров.
* **Цветовая индикация:** Для улучшения читаемости игрового процесса в консоли используется цветовое выделение символов игроков и элементов интерфейса через вспомогательный класс `ConsoleHelper`.

## 4. Архитектура проекта
Код написан с соблюдением принципов **SOLID** и **Clean Code**. Архитектура разделена на независимые модули по принципу единственной ответственности (**SRP**):

* **`Board.cs`**: Логическое представление поля и алгоритм поиска победных комбинаций.
* **`Game.cs`**: Основной игровой цикл, управление очередностью ходов и обработка логики матча.
* **`Player.cs`**: Модель данных, описывающая сущность игрока (имя, символ, количество побед).
* **`SaveManager.cs`**: Абстракция для работы с файловой системой и сериализацией данных.
* **`Menu.cs`**: Реализация пользовательской навигации и управления командами.
* **`ConsoleHelper.cs`**: Инструментарий для централизованного управления цветами и форматированием вывода в консоль.

## 5. Соответствие критериям качества
* **Современный C#**: Использование **Top-level statements** для минимизации шаблонного кода в точке входа (`Program.cs`).
* **Актуальный стек**: Проект базируется на платформе **.NET 9.0**, что позволяет использовать новейшие возможности языка C# 13.
* **Надежность**: Программа корректно обрабатывает исключительные ситуации, предотвращая аварийное завершение при ошибках ввода.
* **Чистота кода**: Понятный нейминг, отсутствие "магических чисел" и следование принципу DRY.
